Recommended Workflow

  1. Open the application and upload a `.pcap`, `.cap`, or `.pcapng` file from the Home page.
  2. Use Basic Information to review decoded packets and inspect packet details in the modal dialog.
  3. Use Protocol Analysis and Traffic Analysis to understand protocol mix, time-based traffic changes, and top peers.
  4. Open Access Map to visualize remote endpoints and geographic lookups when GeoIP data is available.
  5. Use the extraction menus for Web, Mail, FTP, Telnet, client fingerprints, sensitive data, warnings, and recovered files.
  6. Export evidence where needed, including packet-detail PDF output and extracted files from the runtime folders.
The app currently keeps one loaded capture in memory for the active desktop session. Load a new file when you want to switch investigations.

Feature Guide

Data Analysis

  • Basic packet table with protocol and endpoint filtering.
  • Packet-detail modal backed by Scapy decoding.
  • Protocol summary charts and DNS or HTTP endpoint statistics.
  • Traffic charts for relative time, direction, peers, and byte totals.

Data Extraction

  • Session recovery for Web, Mail, FTP, and Telnet traffic.
  • Client information summaries and sensitive-value extraction.
  • Warning views for suspicious or noteworthy capture content.

File Recovery

  • Recover Web, Mail, and FTP transferred files.
  • Recover all detected binary artifacts into runtime export folders.
  • Download renamed files safely through the browser.

Deployment and Packaging

  • Local run: use StartMe.bat to create the virtual environment, install dependencies, and launch the browser.
  • Developer run: use python run.py after installing the requirements.
  • Windows EXE: use build_exe.bat to build dist\PcapAnalyzer.exe with PyInstaller.
  • Repository: public source, help, screenshots, and packaging notes live at {{ github_repository }}.
